Spring Boot Starter 自定义实战

Starter 是 Spring Boot 自动配置的核心载体，理解并掌握 Starter 的开发，是进阶 Spring Boot 的必经之路。本文将手把手带你开发一个完整的自定义 Starter。

1. Starter 是什么

1.1 Starter 的本质

Starter 本质上是一个空的 JAR 包，它的作用是：

  1. 聚合依赖：将相关的依赖打包在一起，简化依赖管理
  2. 自动配置：提供自动配置类，实现”开箱即用”
  3. 约定优于配置：提供合理的默认值，减少配置工作

1.2 官方 Starter 命名规范

示例：

  • spring-boot-starter-web：官方 Web Starter
  • mybatis-spring-boot-starter：MyBatis 第三方 Starter
  • druid-spring-boot-starter：Druid 第三方 Starter

6. 常见问题与解决方案

6.1 自动配置类没有被加载

问题：Starter 引入后，自动配置类没有生效。

排查步骤：

  1. 检查 META-INF/spring/*.imports 文件是否存在
  2. 检查自动配置类的全限定名是否正确
  3. 开启 debug 模式查看条件匹配报告：
  1. 检查是否被 @ConditionalOnXxx 条件排除

6.2 配置属性无法绑定

问题：配置文件中的属性无法绑定到 Properties 类。

解决方案：

  1. 确保 @EnableConfigurationProperties 或 @ConfigurationPropertiesScan 已添加
  2. 检查属性名是否匹配（支持 kebab-case、camelCase、snake_case）
  3. 确保类有无参构造函数或使用 @ConstructorBinding

6.3 Bean 循环依赖

问题：Starter 中的 Bean 出现循环依赖。

解决方案：

  1. 使用 @Lazy 延迟加载：
  1. 使用 ObjectProvider 延迟获取：

6.4 多 Starter 冲突

问题：引入多个类似功能的 Starter 出现冲突。

解决方案：

  1. 使用 @ConditionalOnMissingBean 避免重复注册
  2. 使用 @Primary 指定优先 Bean
  3. 使用 @Qualifier 明确指定注入的 Bean
