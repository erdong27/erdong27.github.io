JVM 监控工具使用指南

发表于2026-04-23|更新于2026-04-23|Java

版本说明

⚠️ 本文基于以下版本编写：

  • JDK: 8 / 11 / 17 / 21
  • JVM: HotSpot
  • 工具: jcmd、jstat、jmap、jstack、VisualVM、JConsole、Arthas、async-profiler

前言

JVM 监控是 Java 后端开发者的必备技能。当线上应用出现内存泄漏、CPU 飙升、线程死锁或频繁 GC 时，能否快速定位问题往往取决于你对监控工具的熟练程度。

本文系统梳理 JVM 生态中的核心监控工具，从 JDK 内置命令行工具到可视化 GUI，再到生产环境首选的 Arthas 和 async-profiler，帮助你建立完整的 JVM 监控知识体系。

一、JDK 内置命令行工具

JDK 的 bin 目录下自带了一系列诊断工具，无需额外安装，是排查问题的第一道防线。

1.1 jps — 查看 Java 进程

1.2 jstat — JVM 统计信息

jstat 用于监控 JVM 的内存、GC、类加载等运行时统计数据。

-gc 输出字段说明：

字段 含义
S0C / S1C Survivor 0/1 区容量 (KB)
S0U / S1U Survivor 0/1 区已使用 (KB)
EC / EU Eden 区容量 / 已使用 (KB)
OC / OU Old 区容量 / 已使用 (KB)
MC / MU Metaspace 容量 / 已使用 (KB)
YGC / YGCT Young GC 次数 / 总耗时 (秒)
FGC / FGCT Full GC 次数 / 总耗时 (秒)
GCT GC 总耗时 (秒)

四、性能剖析神器：async-profiler

async-profiler 是一款低开销的 Java 性能分析工具，支持 CPU、内存分配、锁竞争等多种分析模式。

4.1 特点

  • 低开销：使用 perf_events 和 AsyncGetCallTrace，性能影响极小
  • 无侵入：无需修改代码或添加 JVM 参数
  • 多维度：CPU、alloc、lock、wall 等多种分析模式
  • 火焰图输出：直观展示热点方法

4.4 火焰图解读

火焰图的纵轴表示调用栈深度，横轴表示样本数量（占比）。

  • 宽条 = 热点方法：需要重点关注
  • 颜色无特殊含义：仅用于区分不同方法
  • 从下往上读：底部是调用入口，顶部是被调用的方法

优化方向：

  • 找到最宽的”平顶”，即自耗时最高的方法
  • 关注递归调用和重复计算
  • 检查是否有不必要的同步块

五、工具选型指南

场景 推荐工具 说明
快速查看进程/内存/GC jcmd / jstat 轻量、无依赖
线程死锁/CPU 飙高 jstack / Arthas thread 快速定位问题线程
内存泄漏分析 jmap / Arthas heapdump + MAT 堆转储 + 专业分析工具
线上实时诊断 Arthas 无需重启、功能全面
性能热点分析 async-profiler 低开销、火焰图直观
持续监控告警 Prometheus + JMX Exporter 集成 Grafana 可视化
深度性能调优 JMC + Flight Recorder 事件驱动、细节丰富

7.2 监控指标基线

指标 健康阈值 说明
GC 吞吐量 > 99% GC 时间占比应 < 1%
Full GC 频率 < 1 次/小时 频繁 Full GC 需排查
Young GC 耗时 < 50ms 过长说明 Eden 区过大或对象存活率高
堆内存使用率 < 80% 预留缓冲，避免 OOM
Metaspace 使用率 < 90% 类加载过多或动态生成类泄漏
线程数 < 线程池上限 线程泄漏会导致 OOM

7.3 持续监控方案

总结

JVM 监控工具从简单的命令行到复杂的可视化分析，各有适用场景：

  • 日常巡检：jcmd、jstat 快速查看状态
  • 问题定位：jstack 分析线程，jmap 分析内存
  • 线上诊断：Arthas 是首选，功能全面且安全
  • 性能剖析：async-profiler 生成火焰图，直观高效
  • 持续监控：Prometheus + Grafana 建立可视化监控体系

掌握这些工具，你将能从”凭经验猜问题”进化为”用数据定位问题”，在面对线上故障时游刃有余。
