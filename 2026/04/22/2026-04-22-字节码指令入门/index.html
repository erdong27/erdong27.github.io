版本说明

⚠️ 本文基于以下版本编写：

  • JDK: 8 / 11 / 17
  • JVM: HotSpot
  • 工具: javap (JDK 内置)、ASM、Javassist

前言

作为 Java 开发者，我们每天都在写 .java 文件，但 JVM 执行的并非这些源代码，而是编译后的 .class 文件中的字节码指令。理解字节码，不仅能帮助我们读懂底层原理，还能在排查问题（如 OOM、性能瓶颈）时提供有力支持。

本文将带你入门字节码指令，从基础概念到实战操作，循序渐进。

一、什么是字节码？

1.1 定义

字节码（Bytecode）是一种中间代码形式，由 Java 编译器（javac）生成，存储在 .class 文件中。它是一系列单字节（8位）的操作码（Opcode），每条指令对应 JVM 的一条具体操作。

1.2 为什么学习字节码？

  • 理解语言本质：理解 for 循环、try-catch、Lambda 表达式在底层的实现
  • 性能优化：识别无效对象创建、冗余操作
  • 问题排查：通过字节码分析死锁、内存泄漏等疑难杂症
  • 工具开发：字节码插桩、APM 监控、热修复框架的基石

3.4 对象创建与操作

指令 说明
new 创建新对象
newarray 创建数组
getfield 获取实例字段
putfield 设置实例字段
getstatic 获取静态字段
invokespecial 调用构造函数/私有方法

⚠️ 注意：这里有一个 dup 指令！因为 new 之后对象引用在栈顶，调用构造函数后会消耗掉这个引用，dup 复制了一份用于后续存储。

3.5 方法调用指令

指令 说明 使用场景
invokevirtual 虚方法调用 普通实例方法（动态分派）
invokespecial 特殊方法调用 构造函数、私有方法、父类方法
invokestatic 静态方法调用 静态方法
invokeinterface 接口方法调用 接口实现类的方法
invokedynamic 动态调用 Lambda、函数式接口

七、总结

本文介绍了 Java 字节码的基础知识：

  1. 字节码是什么：JVM 执行的中间代码，由 javac 编译生成
  2. 查看方法：javap 是最常用的工具
  3. 指令分类：加载存储、算术、类型转换、对象创建、方法调用、控制流、异常同步
  4. 实战应用：通过字节码分析代码问题，如无效字符串拼接、过度装箱
  5. 工具生态：ASM、Javassist 等为字节码工程提供基础

掌握字节码，就像拥有了 JVM 的”透视眼”，能帮助我们写出更高效、更可靠的 Java 代码。
